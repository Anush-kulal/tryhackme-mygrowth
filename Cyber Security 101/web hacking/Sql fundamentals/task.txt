-primary key
*It is a column or set of columns that uniquely identifies each row in a table.
*It cannot have multiple colums of having same content.
*It cannot have null values.

-foreign key
*It is a column or set of columns that uniquely identifies each row in a table.

*It cannot have null values.

#questions
1.What type of database should you consider using if the data you're going to be storing will vary greatly in its format?
Answer:Non-relational database

2.What type of database should you consider using if the data you're going to be storing will reliably be in the same structured format?
Answer:relational database

3.In our example, once a record of a book is inserted into our "Books" table, it would be represented as a ___ in that table?

4.Which type of key provides a link from one table to another?
Answer:foreign key

5.which type of key ensures a record is unique within a table?

##Data base and table statements

>CREATE DATABASE <database_name>;
>USE <database_name>;
>SHOW TABLES;
>DROP DATABASE <database_name>;

>CREATE TABLE <table_name> (<column_name> <data_type>, <column_name> <data_type>, ...);
>SHOW TABLES;
>DESCRIBE <table_name>;
It is used to check or understand the table's characteristics.

>ALTER TABLE <table_name> ADD <column_name> <data_type>;
It is used to add a column to the table.

>DROP TABLE <table_name>;

##CRUD operations
CRUD (Create,Read,Update,Delete)

#Create operations
>INSERT INTO <table_name> (<column_name>, <column_name>, ...) VALUES (<value1>, <value2>, ...);

#Read operations

>SELECT * FROM <table_name>;
To Read all the data from the table.

>SELECT <column_name>, <column_name>, ... FROM <table_name>;
To Read specific columns from the table.

>SELECT * FROM <table_name> WHERE <condition>;
To Read specific rows from the table.

#Update operations
>UPDATE <table_name> SET <column_name> = <value>, <column_name> = <value>, ... WHERE <condition>;
To Update specific rows in the table.

#Delete operations
>DELETE FROM <table_name> WHERE <condition>;
To Delete specific rows from the table.
