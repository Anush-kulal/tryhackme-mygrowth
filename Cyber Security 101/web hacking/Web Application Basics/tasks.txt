##HTTP request:Headers and body
#Request headers

*Request Headers allow extra information to be conveyed to the web server about the request. Some common headers are as follows:

Common Request Headers

Request Header|Example|Description
Host|Host: tryhackme.com|Specifies the name of the web server the request is for.
User-Agent|User-Agent: Mozilla/5.0|Shares information about the web browser the request is coming from.
Referer|Referer: https://www.google.com|Indicates the URL from which the request came from.
Cookie|Cookie: user_type=student; room=introtowebapplication; room_status=in_progress|Information the web server previously asked the web browser to store is held in cookies.
Content-Type|Content-Type: application/json|Describes what type or format of data is in the request.

#Request body
*URL encoded
*form data
*JSON
*XML

*URL encoded (application/x-www-form-urlencoded)
name=John&age=25&city=New%20York

*Form data (multipart/form-data)
It allows multiple data blocks.

*JSON (application/json)
{
    "name": "John",
    "age": 25,
    "city": "New York"
}

*XML (application/xml)
<person>
    <name>John</name>
    <age>25</age>
    <city>New York</city>
</person>

##HTTP response:status line and status code
#status line 
-1st line in every http response
*HTTP version
*Status code: 3 digit number
*Reason phrase: human-readable text describing the status code (ok,notfound,internal server error)

Status code and reason phrase
100-199: Informational - Request received, continuing process
200-299: Success - Request was successfully received, understood, and accepted
300-399: Redirection - Further action needs to be taken to complete the request
400-499: Client Error - The request contains bad syntax or cannot be fulfilled
500-599: Server Error - The server failed to fulfill an apparently valid request

100(continue): The request has been received and the client can continue with the request
200(ok): The request has been successfully received, understood, and accepted
201(created): The request has been successfully received, understood, and accepted
204(no content): The request has been successfully received, understood, and accepted
301(moved permanently): The request has been successfully received, understood, and accepted
302(found): The request has been successfully received, understood, and accepted
304(not modified): The request has been successfully received, understood, and accepted
400(bad request): The request has been successfully received, understood, and accepted
401(unauthorized): The request has been successfully received, understood, and accepted
403(forbidden): The request has been successfully received, understood, and accepted
404(not found): The request has been successfully received, understood, and accepted
500(internal server error): The request has been successfully received, understood, and accepted
502(bad gateway): The request has been successfully received, understood, and accepted
503(service unavailable): The request has been successfully received, understood, and accepted
504(gateway timeout): The request has been successfully received, understood, and accepted

##security headers
#security headers are used to protect the web application from security threats

*control-security-policy
- default-src 'self'
- script-src 'self'
- style-src 'self'

Example of csp
Content-Security-Policy: default-src 'self'; script-src 'self'https://trusted.com; style-src 'self'

server -----> client

*strict-transport-security
-HSTS is used to enforce the use of HTTPS
-It is a security header that is used to protect the web application from security threats

Example of HSTS
Strict-Transport-Security: max-age=31536000; includeSubDomains

server -----> client

-It is mainly enforces browser to HTTPs connection, not HTTP.

*X-Frame-Options
-It is used to prevent clickjacking attacks
-It is a security header that is used to protect the web application from security threats

Example of X-Frame-Options
X-Frame-Options: nosniff

nosniff: It is used to prevent the browser from sniffing the content type of the response.
nosniff=no guess

server -----> client
